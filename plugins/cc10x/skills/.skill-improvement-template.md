# Skill Improvement Template

This template provides standard formats for adding missing sections to skills.

## Quick Start Section Format

Add immediately after the Overview section:

```markdown
## Quick Start

[One concrete example showing the skill in action - 3-5 lines]

**Example:**

[Specific code/command/process example that demonstrates the skill]

**Result:**

[Expected outcome or output]
```

## Examples Section Format

Add after Instructions/Process section:

```markdown
## Examples

### Example 1: [Scenario Name]

**Context:** [Brief context]

**Input/Request:** [What user/claude provides]

**Process:** [How skill is applied]

**Output:** [What results]

**Code Example:**
\`\`\`[language]
[concrete code example]
\`\`\`
```

## Troubleshooting Section Format

Add before References section:

```markdown
## Troubleshooting

**Common Issues:**

1. **[Issue Name]**
   - **Symptom**: [What you see or experience]
   - **Cause**: [Why it happens]
   - **Fix**: [How to resolve]
   - **Prevention**: [How to avoid]

2. **[Issue Name]**
   - **Symptom**: [What you see]
   - **Cause**: [Why it happens]
   - **Fix**: [How to resolve]

**If issues persist:**

- Verify functionality analysis was completed first
- Check that all prerequisites are met
- Review skill discovery to ensure correct skill is loaded
- Check orchestrator logs for activation issues
```

## Dependencies/Requirements Section Format

Add after Overview or in dedicated Requirements section:

```markdown
## Requirements

**Dependencies:**

- [Tool/library/requirement name] - [Purpose]
- [Another requirement] - [Purpose]

**Prerequisites:**

- [Knowledge/skill needed]
- [Other prerequisite]

**Tool Access:**

- Required tools: [List if applicable]
- Optional tools: [List if applicable]
```

## Validation Checklist Format

Add before References section (optional but recommended):

```markdown
## Validation Checklist

Before using this skill, verify:

- [ ] [Check 1 - specific to skill]
- [ ] [Check 2 - specific to skill]
- [ ] [Check 3 - specific to skill]

After applying this skill, verify:

- [ ] [Verification 1]
- [ ] [Verification 2]
```

## Notes

- Preserve existing functionality-first approach
- Maintain consistency with cc10x patterns
- Keep progressive disclosure (supporting files) but add inline examples
- Enhance, don't replace existing content
