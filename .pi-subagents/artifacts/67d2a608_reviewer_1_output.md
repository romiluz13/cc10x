### Issues found

- **MEDIUM** — Coverage gate changed from an unconditional v11 floor to fully opt-in. v11 `test-driven-development/SKILL.md` ("Coverage Threshold (Project Default)") stated an unconditional 80% target described as "a floor, not a ceiling." Current `building/SKILL.md:137` and `component-builder.md:87`: "If `coverage-thresholds.json` exists... Below thresholds → FAIL. If no thresholds file, skip coverage check." Since the template lives in `plugins/cc10x/templates/coverage-thresholds.json` and must be manually copied in, most projects get **zero** coverage enforcement by default post-refactor, versus always-on advisory enforcement in v11. Deliberate design choice (opt-in vs hardcoded), but it is a real behavior regression for any project that doesn't install the template.
- **MEDIUM** — "Option Zero" (config-only-fix check before proposing SDK replacement/migration; "who reads env at import time") from v11 `debugging-patterns/SKILL.md` Phase 1 point 6 is **completely gone** — confirmed via `grep -rn "Option Zero\|reads env at import\|module import" plugins/` returning nothing anywhere in the plugin tree. This is a concrete, load-bearing diagnostic technique (prevents wasted large migrations for what's actually a wiring bug), not decorative prose. No replacement found in `debugging/SKILL.md`, `root-cause-playbooks.md`, or `bug-investigator.md`.
- **MEDIUM** — "Context Retrieval (Large Codebases)" 3-cycle DISPATCH/EVALUATE/REFINE technique, present in v11 `bug-investigator.md` step 3, is gone from the current `bug-investigator.md` entirely; only found in `plugins/cc10x/agents/planner.md` now. For an ad-hoc DEBUG invocation that never goes through PLAN, the investigator has lost its own scoped-context-narrowing technique for multi-file bugs.
- **MEDIUM** — v11 `code-generation/SKILL.md`'s "Rule of three" (When to Abstract table), "Universal Questions" pre-write checklist, and the "Grep-check 'implement it properly'" scope-creep detector (grep for real callers before honoring vague "make it complete" review language) are all absent from current `building/SKILL.md` with no equivalent found anywhere (`grep -rn "Rule of three|When to Abstract|Universal Questions|Grep-check" plugins/cc10x/skills/` → no hits). Decision Checkpoints/Scope Escalation partially cover scope-creep at a coarser "file count" granularity, but the specific "0 callers found — remove, don't build out" evidence pattern is lost.
- **LOW** — v11 bug-investigator's "Anti-loop rule" sentence ("Analysis without action is a stuck signal...") is dropped with no direct replacement; overlaps substantially with the Causal Chain Gate/Hypothesis Confidence Scoring that remain, so functional impact is small.
- **LOW** — The "scar-note" dated-gate-justification convention from v11 TDD skill is gone from building/debugging entirely (one unrelated mention survives in `exploration/SKILL.md`). Meta-discipline, not a functional gate.
- **LOW** — `skills/building` has no `evals/` pressure-test directory while `skills/debugging` has 3 (outage/sunk-cost/authority). Building is equally exposed to "just ship it" pressure but has no adversarial eval coverage validating the Iron Law holds under pressure.

- Rationalization tables and BAD/GOOD examples were trimmed, not deleted — still present in condensed form in `building/SKILL.md`, `test-data-and-mocks.md`, and `debugging/SKILL.md`.

### Verdict

Build + Debug are **publish-ready with one pre-release fix recommended**: restore or explicitly document that coverage enforcement is now opt-in (currently a silent behavior change from v11's always-on floor — at minimum this should be called out in a changelog/README since it changes default rigor for any project that doesn't copy the template). The 5 other MEDIUM/LOW items (Option Zero, bug-investigator's own context-retrieval cycle, Rule-of-Three/Universal-Questions/scope-creep-grep from code-generation, the anti-loop sentence, scar-notes, and building's missing eval coverage) are real but non-blocking losses of narrow, mostly-advisory techniques — they reduce depth in edge cases rather than removing a hard gate. All 6 explicitly-named "keep inventory" innovations (clean-baseline diff, per-phase BASE sha, SCOPE_INCREASES escalation, Feedback-Loop-FIRST ladder, blast-radius scan, verify-before-implement dispute) are intact and, in the case of dispute-adjudication and baseline diffing, more rigorously specified than the compressed line counts would suggest — the aggressive line-count compression in `component-builder.md`/`bug-investigator.md` did not come at the cost of these load-bearing gates.
