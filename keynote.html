Part I — Harness Engineering

What is a harness?

The harness is everything around the LLM call —
the scaffolding that determines what the model sees,
what it's allowed to do, and how its output is verified.
👁
Context control
What the model sees. Which files. Which history. Which constraints. Curated, not dumped.
🚧
Permission surface
What the model can touch. WRITE agents vs READ-ONLY agents. Roles enforced structurally, not by asking nicely.
✅
Verification gates
Evidence before advancement. No phase exits without proof. "exit 0 or it didn't happen."

The loop
planner writes plan → gap-reviewer challenges from fresh context → if blocking findings: re-plan → re-review

max 2 review passes

If pass 1 succeeds: prune re-plan + pass 2 (unused branches deleted).
If pass 2 finds issues: stop with clarification gate.
Why it works
Anti-anchoring by design — the gap-reviewer loads ZERO memory, ZERO skills. It has never seen the planner's rationale. It reviews the plan against the codebase, not against the planner's confidence.

Agreement-first — planner returns NEEDS_CLARIFICATION on ambiguity. Never treats its own defaults as approved.

Doubt-Driven Development — for high-stakes decisions, an in-flight adversarial reviewer challenges the design BEFORE the plan is finalized.

The loop
Feedback loop gate (MANDATORY): build repro signal BEFORE any hypothesis. No loop → fail closed.

Hypothesis loop: generate 3-5 ranked hypotheses up front → test → if all fail: NEEDS_EXTERNAL_RESEARCH

Fix loop: RED (failing test) → GREEN (minimal fix) → blast radius scan → verify

3 failed attempts then: research or BLOCKED
Why it works
No-Loop-No-Hypothesis gate — you can't guess without a repro. Prevents pattern-matching death spirals.

Causal chain gate — must explain the full chain from trigger to symptom. "Somehow X leads to Y" is a gap, not an answer. Predictions catch symptom fixes.

Debug memory — every failed hypothesis stored as [DEBUG-N] in activeContext.md. The system remembers what didn't work.

6 passes: Security → Performance → Quality → Friction → Plan Validity → Spec Compliance

Quantitative scoring: HARD/SOFT signals, min(HARD) capped by avg(SOFT)-10. Confidence ≥80 to report.
The loop
No remediation loop — REVIEW is advisory. It never creates REM-FIX tasks or mutates code.

If verdict is CHANGES_REQUESTED: router offers "Start BUILD to fix (Recommended)" — a transition, not an automatic action.

Zero-finding gate: zero findings on non-trivial change → confidence capped at 70. Must produce positive assertions with file:line evidence. No rubber-stamps.
ORIENT (read-only)
Zero agents. Zero tasks. Zero artifacts.

The router answers inline — maps modules, traces call hierarchy via LSP, explains in the project's own vocabulary.

"Help me understand this code" never falls through to BUILD and spawns a write agent. ORIENT precedes DEFAULT.

Why: orientation is the most common request. Routing it to BUILD would mean a builder modifying code you just wanted explained.
The gate that catches green lies

"Tests passed" is not evidence.
Exit 0 is evidence.

False-green patterns — auto-flagged
The verifier greps changed test files for tests that pass while proving nothing:

getByTestId('…-mock') — asserting the mock, not the behavior
as any · as unknown — schema-incomplete mocks
.find() · readFileSync — DB-bypass, not the public interface
setTimeout( · sleep( — arbitrary waits, not condition-waits

A hit doesn't auto-fail — but that scenario can't count as PASS on that test's strength alone.
Quote-the-line gate
Every finding at confidence ≥80 MUST include a verbatim file:line quote. No quote → auto-demoted below the reporting bar.

Then the verifier independently re-reads the exact file:line and marks it:
validated: true  validated: false → dropped

A hallucinated critical finding cannot waste a fix cycle. The verifier kills it before it gates.

"Should work now" → RUN the command. "Should" is not evidence.
"I'm confident it works" → Confidence is not evidence. Exit code 0 is evidence.

Principles you can take home today

The 5 laws of harness + loop engineering.

1.
One entry point, one owner. The harness owns orchestration state. Agents execute; they never route.
2.
Gates are evidence, not time. A phase ends when proof exists — not when N seconds pass or the model says it's done. Artifact integrity gate blocks by default; other gates audit by default, upgradeable to block.
3.
Loops have a ceiling. Unbounded loops drift. Hard limit + human checkpoint at N is a design constraint, not a workaround.
4.
Expertise is injected, not assumed. The harness decides what skills are in scope. Models don't self-select their constraints.
5.
Memory closes the loop across sessions. What failed once is a gotcha forever. The system gets smarter; the model doesn't have to.

The scar: one real dispatch hit 42k chars — 99% pasted history. The orchestrator that pastes the prior conversation into every child rots its own context. The fix became law: dispatch by reference, not by blob — write the diff to disk, pass the path, never the body.
# The minimum viable harness def harness(request): intent = route(request) # deterministic context = load_memory() # structured skills = inject_hints(intent) # contextual for phase in intent.chain: result = run_agent( phase, context, skills ) if not gate_passes(result): return remediate(result) # loop persist_memory(result) # closes loop return result
route() — keyword signal → deterministic workflow. No model judgment at this layer.
gate_passes() — structured contract validation. YAML, not prose. Schema, not vibes.
remediate() — creates a scoped fix task with evidence. Bounded. Re-enters the loop with new context.
persist_memory() — gotchas, patterns, decisions. The loop learns even if the model doesn't.
