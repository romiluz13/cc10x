Part I — Harness Engineering

What is a harness?

The harness is everything around the LLM call —
the scaffolding that determines what the model sees,
what it's allowed to do, and how its output is verified.
👁
Context control
What the model sees. Which files. Which history. Which constraints. Curated, not dumped.
🚧
Permission surface
What the model can touch. WRITE agents vs READ-ONLY agents. Roles enforced structurally, not by asking nicely.
✅
Verification gates
Evidence before advancement. No phase exits without proof. "exit 0 or it didn't happen."

Fail-closed gates

Part II — Loop Engineering

Not retries.
Convergence loops.

✗
Not retries — hoping the model gets it right on attempt N
✗
Not polling — running the same check until it passes
✓
Evidence-driven iteration — each loop consumes findings from the prior cycle as structured input
✓
Bounded convergence — hard limits prevent infinite drift. At N cycles: human checkpoint.

Loop Engineering — CC10x as living proof

Three loops.
All bounded. All evidence-gated.

🔁
Remediation loop
reviewer/hunter finds issues →
router creates REMFIX task →
builder fixes →
re-review / re-verify

max 3 cycles  then: human checkpoint
🐛
Debug loop (Reflexion-style)
LOG FIRST → reproduce → hypothesize → fix

Each failed attempt is stored in patterns.md as a gotcha.

verbal memory  improves next session
📋
Plan review loop
planner writes plan →
plan-gap-reviewer anti-anchors →
REPLAN if blocking findings

fresh context  no rubber-stamping

Every loop has an exit condition based on evidence, not time or count.
The gate that triggers the loop is the same gate that closes it.

Principles you can take home today

The 5 laws of harness + loop engineering.

1.
One entry point, one owner. The harness owns orchestration state. Agents execute; they never route.
2.
Gates are evidence, not time. A phase ends when proof exists — not when N seconds pass or the model says it's done.
3.
Loops have a ceiling. Unbounded loops drift. Hard limit + human checkpoint at N is a design constraint, not a workaround.
4.
Expertise is injected, not assumed. The harness decides what skills are in scope. Models don't self-select their constraints.
5.
Memory closes the loop across sessions. What failed once is a gotcha forever. The system gets smarter; the model doesn't have to.
Build your own

You don't need CC10x.
You need the pattern.

# The minimum viable harness def harness(request): intent = route(request) # deterministic context = load_memory() # structured skills = inject_hints(intent) # contextual for phase in intent.chain: result = run_agent( phase, context, skills ) if not gate_passes(result): return remediate(result) # loop persist_memory(result) # closes loop return result
route() — keyword signal → deterministic workflow. No model judgment at this layer.
gate_passes() — structured contract validation. YAML, not prose. Schema, not vibes.
remediate() — creates a scoped fix task with evidence. Bounded. Re-enters the loop with new context.
persist_memory() — gotchas, patterns, decisions. The loop learns even if the model doesn't.
